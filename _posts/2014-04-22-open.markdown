Wherever I see, I feel that there is something broke. I can't always pin point the issue but I can't shake the feeling. But I was able to pin point in case of open source projects. 

How many of you are working on open source projects? I mean, dedicatedly. How many of you feel that trying out new project to work on is always a headache. You should have project and its dependencies setup on development system. There are always steps that you need to follow. One for each project and the steps may vary with operating system, sometimes with architecture of machine. 
Even if I am able to set it up successfully, it takes a while to understand complete code flow. It gets worse with different standards and technologies followed and used by each project. For each project, the community around it is the biggest asset and also biggest problem, they want to maintain status-quo and get comfortable with existing process. 

We have so many projects available as open source. Thanks to git and github, we have one common repository for most of the projects ( at least the mirrors for rest ). Whole process is setup in simple interface for the developers to use. 

For a new developer to get into open source, these things are , major barriers. For effective utilization of developer resources world wide, we should be following standard way of doing things. npm is a good example of the same but that has its own limitations. 

For addressing the issue of set-up, maintainers should provide snapshot of working VM which can be directly downloaded and used or they can create a container ( like docker ) which can be used directly by developers without worrying about setting up. For setting up the coding standards, I suggest that everyone should agree on single standard for open source projects, which will be evolving over time.
